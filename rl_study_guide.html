Origin & History of RL
▼

RL has roots in psychology (trial-and-error learning), control theory, and operations research. It emerged as a formal ML subfield in the 1980s–90s.

1950s: Minsky's SNARCs, Shannon's chess — earliest experiments
1957: Bellman formulates Dynamic Programming
1960s: Samuel's checkers player uses self-play RL
1989: Watkins proposes Q-Learning
1992: Tesauro's TD-Gammon beats world-champion backgammon players
2013: DQN (DeepMind) plays Atari — deep RL begins
2016: AlphaGo defeats Lee Sedol — milestone moment
KEY INSIGHT
RL is inspired by how animals learn from consequences. A dog learns to sit for a treat — same idea: action → reward signal → policy update.

Elements of RL
▼

Every RL system is defined by five core components:

Agent — the learner/decision-maker
Environment — everything the agent interacts with
State (s) — current situation of the environment
Action (a) — what the agent chooses to do
Reward (r) — scalar feedback signal after each action
Policy (π) — mapping from states to actions (what to do)
Value Function — expected long-term reward from a state
Model (optional) — agent's internal model of environment dynamics
🚧
Limitations & Scope of RL
▼

RL is powerful but has real-world constraints you must know for exams and practice.

Sample inefficiency — needs millions of interactions; expensive in real world
Reward design is hard — poorly specified rewards cause reward hacking
Credit assignment problem — which past action caused this reward?
Exploration vs Exploitation — try new things vs use known-good actions
Non-stationarity — environment or policy may shift over time
Partial observability — agent may not see the full state
SCOPE
RL works best in well-defined environments with clear reward signals — games, robotics, recommendation systems, resource management.

Deep Reinforcement Learning
▼

Deep RL = RL + Deep Neural Networks as function approximators. This overcomes the curse of dimensionality in traditional tabular RL.

Problem with tabular RL: Need a table entry for every state-action pair — impossible for Atari (10^70 states)
Solution: Use a neural network to approximate Q(s,a) or π(a|s)
DQN (2013): CNN processes raw pixels → Q-values for each action
Key tricks: Experience Replay + Target Network → stable training
AlphaGo, AlphaZero, OpenAI Five, ChatGPT (RLHF) — all deep RL
Input (pixels/state) → CNN/MLP → Q-values for each action Policy: π(s) = argmax_a Q(s,a)

Introduction to MDP
▼

A Markov Decision Process is the mathematical framework for sequential decision-making under uncertainty.

MDP = (S, A, P, R, γ) S = set of states A = set of actions P(s'|s,a) = transition probability R(s,a,s') = reward function γ ∈ [0,1] = discount factor
MARKOV PROPERTY
The future depends only on the present state, not the history: P(sₜ₊₁|sₜ,aₜ,...s₁,a₁) = P(sₜ₊₁|sₜ,aₜ)
Finite MDP: finite S and A — tabular solution possible
Episodic: has terminal state (game ends)
Continuing: no terminal state, runs forever (use discounting)
γ close to 0 → myopic (values immediate rewards); γ close to 1 → far-sighted
💎
State & Action Value Functions
▼

Two critical functions measure the "goodness" of being in a state or taking an action.

State Value Function V^π(s): V^π(s) = E_π[Gₜ | sₜ=s] = E_π[Σ γᵏ rₜ₊ₖ₊₁ | sₜ=s] Action Value Function Q^π(s,a): Q^π(s,a) = E_π[Gₜ | sₜ=s, aₜ=a] Return Gₜ = rₜ₊₁ + γrₜ₊₂ + γ²rₜ₊₃ + ...
V(s): expected return starting from state s, following policy π
Q(s,a): expected return taking action a in state s, then following π
Relationship: V^π(s) = Σ_a π(a|s) · Q^π(s,a)
Q is more useful in practice — can derive policy directly: π(s) = argmax_a Q(s,a)
🔔
Bellman Expectation Equations
▼

The Bellman equations express value functions recursively — the value of a state equals immediate reward plus discounted value of the next state.

Bellman Expectation for V^π: V^π(s) = Σ_a π(a|s) Σ_{s'} P(s'|s,a)[R(s,a,s') + γ·V^π(s')] Bellman Expectation for Q^π: Q^π(s,a) = Σ_{s'} P(s'|s,a)[R(s,a,s') + γ·Σ_{a'} π(a'|s')·Q^π(s',a')]
INTUITION
Think recursively: "Value of being here = reward I'll get now + how good the place I end up will be." This forms a system of linear equations solvable for small MDPs.
Basis for Policy Evaluation (computing V^π for a given policy)
Can be solved exactly for small tabular MDPs
For large/continuous spaces → approximate with neural networks
🏆
Optimality & Bellman Optimality
▼

An optimal policy π* achieves the maximum expected return from every state. The optimal value functions satisfy the Bellman Optimality Equations.

Bellman Optimality for V*: V*(s) = max_a Σ_{s'} P(s'|s,a)[R(s,a,s') + γ·V*(s')] Bellman Optimality for Q*: Q*(s,a) = Σ_{s'} P(s'|s,a)[R(s,a,s') + γ·max_{a'} Q*(s',a')] Optimal Policy: π*(s) = argmax_a Q*(s,a)
KEY DIFFERENCE
Bellman Expectation uses Σ over π(a|s) (average over policy). Bellman Optimality uses max over actions — greedy with respect to optimal Q.
For finite MDPs, optimal policy always exists and is deterministic
Value Iteration: repeatedly apply Bellman optimality update → converges to V*
Policy Iteration: evaluate policy → improve policy → repeat → converges to π*
📐
Policy Gradient Methods
▼

Instead of learning value functions, Policy Gradient methods directly optimize the policy π_θ (parameterized by θ) using gradient ascent on expected return.

Objective: J(θ) = E_π[Gₜ] Policy Gradient Theorem: ∇_θ J(θ) = E_π[∇_θ log π_θ(a|s) · Q^π(s,a)] REINFORCE update: θ ← θ + α · ∇_θ log π_θ(aₜ|sₜ) · Gₜ
Works for continuous action spaces — unlike value-based methods
Can learn stochastic policies (sometimes optimal, e.g., in poker)
log π term: increase probability of actions that led to high return
Problem: very high variance — needs many samples
📉
Reducing Variance in Policy Gradients
▼

High variance in policy gradient estimates makes training unstable. Several techniques reduce variance while keeping the estimator unbiased.

Baseline subtraction (unbiased variance reduction): ∇_θ J(θ) = E_π[∇_θ log π_θ(a|s) · (Q^π(s,a) - b(s))] Most common baseline: b(s) = V^π(s) → gives Advantage A(s,a): A(s,a) = Q^π(s,a) - V^π(s) Actor-Critic update: θ ← θ + α · ∇_θ log π_θ(a|s) · A(s,a)
Baseline: subtract expected value → reduces variance, doesn't change gradient mean
Advantage function A(s,a): "how much better is this action than average?"
Actor-Critic: Actor = policy π_θ; Critic = value function V_w (reduces variance)
PPO, A3C, A2C — modern algorithms all use advantage estimation
GAE (Generalized Advantage Estimation) — exponential average of TD errors
EXAM TIP
A(s,a) > 0 → action was better than average → increase its probability. A(s,a) < 0 → worse than average → decrease its probability.

CLICK TO ANSWER
Q1. The Markov Property states that the next state depends on:
Markov property: future is conditionally independent of history given the present. P(s'|s,a) is all you need — no memory of past states required.
Q2. The Bellman Optimality equation uses _____ over actions, unlike the Bellman Expectation equation which uses _____.
Bellman Expectation averages over the policy (Σ_a π·Q). Bellman Optimality takes the greedy max — giving us the best possible value achievable.
Q3. In Actor-Critic methods, the CRITIC's role is to:
The Critic estimates V(s) or Q(s,a) and provides the advantage signal A(s,a) = Q - V to the Actor, dramatically reducing policy gradient variance.
Q4. What is the relationship between V^π(s) and Q^π(s,a)?
V^π(s) is the expected value when averaging over all actions weighted by the policy π. If you fix the action to a specific a, you get Q^π(s,a). V is the policy-weighted average of Q.

Control Learning Algorithms Overview
▼

Control learning = finding the optimal policy through interaction. Two main families:

PropertyOn-Policy (SARSA)Off-Policy (Q-Learning)
Learns fromCurrent policy's actionsCan learn from any actions
Update usesNext action actually taken (a')Best possible next action (max)
Exploration safetySafer (knows its own behavior)Can be aggressive
ConvergenceConverges to π (not π*)Converges to π* directly
Use caseSafe environments, cliff walkingOptimal policy, most RL
TEMPORAL DIFFERENCE (TD)
Both SARSA and Q-learning are TD methods — they learn from incomplete episodes by bootstrapping: updating value estimates using other estimated values.
🔍
Q-Learning
▼

Q-Learning (Watkins, 1989) is an off-policy TD control algorithm that directly approximates the optimal action-value function Q*.

Q-Learning Update Rule: Q(s,a) ← Q(s,a) + α[r + γ·max_{a'} Q(s',a') − Q(s,a)] Where: α = learning rate (step size) r = immediate reward γ = discount factor max_{a'} Q(s',a') = best Q-value in next state [...] = TD error (δ)
Off-policy: behavior policy (ε-greedy) ≠ target policy (greedy)
TD error δ = r + γ·max Q(s',a') − Q(s,a) — the "surprise" signal
Converges to Q* under standard stochastic approximation conditions
ε-greedy exploration: ε probability random action, (1-ε) greedy
Simple and effective for small discrete state-action spaces
MEMORY TRICK
Q-learning update: "Current Q + learning rate × (Target − Current)" — just like gradient descent, moving Q toward the bootstrap target.
🌊
SARSA — On-Policy TD Control
▼

SARSA (State-Action-Reward-State-Action) is on-policy: it updates Q using the action actually taken by the current policy.

SARSA Update Rule: Q(s,a) ← Q(s,a) + α[r + γ·Q(s',a') − Q(s,a)] Note: a' is sampled from π (e.g., ε-greedy), NOT max Name: uses the quintuple (sₜ, aₜ, rₜ₊₁, sₜ₊₁, aₜ₊₁) for each update
Key difference from Q-learning: uses Q(s', a') not max Q(s', ·)
On-policy → converges to the optimal ε-greedy policy (not exact optimal)
Safer in stochastic, dangerous environments (avoids cliff edges)
Famous example: Cliff Walking — SARSA avoids cliff, Q-learning walks near it
λ
SARSA(λ) — Eligibility Traces
▼

SARSA(λ) extends SARSA with eligibility traces — a memory mechanism that assigns credit to recently visited states.

Eligibility Trace update: e(s,a) ← γ·λ·e(s,a) + 1 [for visited (s,a)] e(s,a) ← γ·λ·e(s,a) [for other (s,a)] TD error: δₜ = rₜ₊₁ + γ·Q(sₜ₊₁,aₜ₊₁) − Q(sₜ,aₜ) Q update for ALL (s,a): Q(s,a) ← Q(s,a) + α·δₜ·e(s,a) Special cases: λ=0 → SARSA (one-step TD) λ=1 → Monte Carlo (full episode)
λ (lambda): decay parameter for traces — how far back to assign credit
Eligibility trace = recency × frequency — recent and frequent states get more credit
Bridges TD(0) and Monte Carlo — uses n-step returns efficiently
Faster learning by propagating reward backward through recent states
λ ∈ (0.8, 0.95) often works well in practice
INTUITION
You scored a goal. Eligibility traces credit not just the last kick, but all your recent moves — the run, the pass, the positioning. More recent actions get more credit (decayed by γλ).
🤖
DQN — Deep Q-Network
▼

DQN (Mnih et al., DeepMind 2013/2015) combines Q-learning with deep neural networks to handle high-dimensional inputs like raw pixels.

DQN Architecture: Input: raw pixels (84×84×4 stacked frames) → 3 Conv layers (feature extraction) → 2 FC layers → Output: Q(s,a) for each discrete action a Loss function: L(θ) = E[(yⱼ − Q(sⱼ,aⱼ;θ))²] Target yⱼ: yⱼ = rⱼ (if terminal) yⱼ = rⱼ + γ·max_{a'} Q(sⱼ',a';θ⁻) (otherwise) θ⁻ = frozen target network weights (updated every C steps)
Key Innovation 1 — Experience Replay: store (s,a,r,s') tuples in replay buffer D; sample random mini-batches → breaks temporal correlation, improves sample efficiency
Key Innovation 2 — Target Network θ⁻: separate frozen network for computing targets; updated every C steps → stabilizes training (prevents chasing a moving target)
Played 49 Atari games at human/superhuman level from raw pixels + reward only
Extensions: Double DQN, Dueling DQN, Prioritized Experience Replay, Rainbow
WHY TWO NETWORKS?
Without a target network, the TD target changes at every step (like chasing your own shadow). Freezing θ⁻ for C steps gives stable learning targets.

⚡ Quick Check — CO3
CLICK TO ANSWER
Q1. SARSA is called an on-policy method because:
On-policy = the learning policy IS the behavior policy. SARSA uses the actual next action a' (from ε-greedy) in its update. Q-learning instead uses max Q(s',·) — off-policy.
Q2. In SARSA(λ), when λ=0, the algorithm reduces to:
λ=0: traces decay immediately → only immediate transition used → standard SARSA / TD(0). λ=1: traces persist for full episode → Monte Carlo returns.
Q3. DQN's "Experience Replay" serves what primary purpose?
Sequential experiences are highly correlated — bad for SGD (assumes i.i.d. data). Replay buffer stores transitions and random mini-batch sampling breaks this correlation, stabilizing training.
Q4. Why does DQN use a separate "target network" θ⁻?
Without a frozen target network, every weight update changes both the prediction AND the target — like chasing your own shadow. θ⁻ is frozen for C steps, giving stable Bellman targets to train toward.
